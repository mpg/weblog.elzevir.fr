TeX Live 2010 gelée

with 3 comments

Cette nuit a eu lieu la dernière mise à jour de TeX Live 2010. D’ici quelques jours (temps de propagation aux miroirs) tlmgr update --all vous répondra systématiquement qu’aucune mise à jour n’est disponible, même si des paquets ont été mis à jour sur le CTAN, et ce sera le cas jusqu’à la sortie de TeX Live 2011.

Soyons précis : TeX Live 2010 est gelée, TeX Live 2011 ne l’est pas encore. Les paquets mis à jour sur le CTAN continuent d’arriver dans TeX Live, seuls deux points changent :

  • les mises à jour ne sont plus propagées sur tlnet, le dépôt de paquet TeX Live principal ; en revanche elles seront bientôt propagées sur tlprestest, le dépôt de paquets temporaire de la phase de test, qui est en train d’être mis en place ;
  • alors que les mises à jour sont habituellement systématiques, elles seront désormais faites avec plus de circonspection afin d’éviter d’inclure dans le DVD des versions boguées de paquets importants.

Soyons encore plus précis : une partie de TeX Live 2011 est d’ores et déjà (presque) gelée. Il s’agit des programmes exécutables compilés (pdfTeX, XeTeX, LuaTeX, dvips, dvipdfm, synctex, etc.) dont les sources finales (sauf corrections de bogues) sont en principe déjà incluses. En effet, construire les 21 versions des exécutables pour les nombreuses plate-formes supportées n’est pas une mince affaire pour les nombreux volontaires impliqués. Obtenir les versions finales des binaires sera une étape importante dans le calendrier.

L’autre étape importante, qui est traitée en parallèle, est la mise à jour de la documentation TeX Live puis de ses traductions. Pendant ce temps, les mises à jour des paquets ordinaires continueront comme indiqué ci-dessus, puis un gel complet interviendra probablement quelques jours avant la fabrication des images finales pour le DVD, ce qui est (pour l’instant) prévu pour début juillet.

L’infrastructure TeX Live n’ayant pas connu d’évolution trop importante cette année, il devrait être possible de mettre à jour une installation TeX Live 2010 vers 2011 sans réinstallation complète, au moins sous Unix. À suivre.

  2. La commande tlmgr opère toujours sur l’installation dont fait partie le programme tlmgr appelé. Le moyen le plus simple pour choisir l’installation à supprimer est donc d’appeler tlmgr avec son chemin complet, par exemple /usr/local/texlive/2009/bin/i386-linux/tlmgr uninstall.

    En fait, sous Unix, à moins d’avoir installé des liens symboliques dans les répertoires standard avec l’option d’installation correspondante ou la commande tlmgr path add, le moyen de plus simple et le plus sûr de supprimer une installation est encore un bon vieux rm -r /usr/local/texlive/2009.
