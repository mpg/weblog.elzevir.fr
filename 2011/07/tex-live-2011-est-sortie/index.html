TeX Live 2011 est sortie

without comments

La version 2011 de TeX Live, la distribution TeX complète, libre et multiplateforme, a été annoncée officiellement cette nuit. Examinons les quelques nouveautés et les procédures de mise à jour.

Nouveautés

Cette mouture n’apporte pas de gros changements, surtout pour ceux qui utilisaient régulièrement tlmgr pour mettre à jour leurs paquets. La liste des nouveautés dans la doc est courte, la principale nouveauté étant l’arrivée de Biber, un successeur de BibTeX prévu pour fonctionner avec BibLaTeX. Les autres nouveautés sont les mises à jour des programmes, par exemple LuaTeX 0.70.1 ou encore MetaPost 1.504, qui n’utilise plus de fichiers .mem (équivalent des fichiers de format .fmt de TeX). Certains programmes sont plus rapides, comme updmap ou tlmgr qui devrait charger la base de donnée des paquets 3 fois plus vite qu’avant.

Mise à jour depuis la version pretest

Si vous aviez installé la version 2011-pretest, tout ce que vous avez à faire pour passer à la version 2011 est de changer l’adresse du dépôt par défaut :

tlmgr option repository ctan

pour utiliser le dépôt par défaut, qui passe par mirror.ctan.org pour sélectionner un miroir automatiquement, ou :

tlmgr option repository http://hote.domaine/chemin/vers/tlnet

pour choisir vous-même un miroir parmi ceux listés ici.

Ensuite, vous pourrez comme d’habitude utiliser

tlmgr update --all -self

pour mettre à jour. (Il est parfaitement normal qu’il n’y ait pas de mises à jour immédiatement, voir ci-dessous.)

Mise à jour depuis TeX Live 2010

Depuis toujours la procédure standard pour installer une nouvelle version de TeX Live consiste à… installer la nouvelle version. Contrairement à un mythe répandu, il ne s’agit pas de désinstaller l’ancienne version puis d’installer la nouvelle : il s’agit d’installer la nouvelle, puis, éventuellement, plus tard, une fois qu’on est sûr que tout va vraiment bien avec la nouvelle mouture, si on a besoin d’espace disque, de désintaller l’ancienne. Il est parfaitement possible de faire cohabiter plusieurs versions de TeX Live sur une même machine (à l’instant où j’écris ces lignes, j’en ai 8 versions sous la main : de 2005 à 2011, avec deux variantes pour 2010 et 2011), de même qu’on peut faire cohabiter MiKTeX et TeX Live sur une machine Windows.

Ceci étant posé, il est possible depuis cette année, sur les machines Unix, d’utiliser l’installation existante comme point de départ pour migrer vers la nouvelle version et des instructions sont fournies. Cette procédure a été peu testée, des problèmes imprévus peuvent survenir ; elle ne devrait donc être utilisée que par des personnes prêtes à gérer les éventuels problèmes et à les signaler sur la liste TeX Live.

La raison pour laquelle cette procédure ne fonctionne pas encore sous Windows est la gestion des différentes entrées dans la base de registres (associations de fichiers, désinstallateur, …) et le menu démarrer.

Reprise des mises à jour

Pendant encore quelques jours, tlnet restera figé afin de laisser les miroirs du CTAN se stabiliser et de traiter tranquillement les éventuels problèmes. Ne vous étonnez donc pas si tlmgr ne propose aucune mise à jour. Pendant ce temps, nous travaillons à empaqueter les mises à jour de paquets depuis le CTAN (actuellement 90 paquets en retard, sans compter les nouveaux paquets) ; il faudra probablement quelque temps pour combler ce retard qui est parfaitement normal. Néanmoins, si quelques semaines après le dégel de tlnet vous constatez encore des paquets manquants ou en retard, n’hésitez pas à les signaler sur la liste TeX Live.
